# assets/

Public assets referenced from the project README and other docs.
Everything here is committed (unlike `web/public/live2d/`, which is
user-supplied and gitignored).

## Layout

- `screenshots/` — UI screenshots used in the project README.
- `demo/` — short GIFs or MP4 demos.
- `logo/` — project mark / wordmark assets if any.

## Adding a screenshot

1. Drop the file under `assets/screenshots/<feature>.png`. Prefer PNG
   for static UI, WebP if you need transparency at smaller sizes.
2. Reference it from the README with a relative path:
   `![Avatar window](assets/screenshots/avatar.png)`.
3. Keep individual files under ~500 KB. If you need bigger
   (e.g., a 4K Live2D shot), host it externally and link to it
   instead of committing.

## Adding a demo GIF

The same rules apply. Try to keep GIFs under 5 MB; the GitHub README
renderer downloads every embedded asset on every view. If a demo
needs to be longer, host on YouTube and embed the thumbnail.

## What does **not** go here

- **User-supplied Live2D models.** Those go under
  `web/public/live2d/models/<name>/` and are gitignored.
- **Audio samples generated by TTS sidecars.** Those are scratch
  outputs and go under `tts_samples/` (gitignored).
- **Live2D Cubism SDK runtime files** (`live2d.min.js`,
  `live2dcubismcore.min.js`). Those are subject to Live2D Inc.'s
  distribution terms and live under `web/public/live2d/`.
